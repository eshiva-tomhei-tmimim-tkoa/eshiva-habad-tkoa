## 1. О чём проект

Редизайн сайта **Ешивы «ХаБаД Ткоа»** (Yeshiva Chabad Tkoa) — учебного заведения
в посёлке Ткоа (Гуш-Эцион, ~25 км от Иерусалима, Израиль).

Ешива основана в 2022 году как **универсальное учебное заведение** для молодых
людей 16–17 лет из стран СНГ, многие из которых после событий 2022 года оказались
в Израиле без знания языка, профессии и иногда без аттестата. Идея сайта —
показать сочетание четырёх линий программы: **Тора и хасидут, профессия, иврит и
адаптация, опытные наставники**.

Сайт — на русском языке (основной), с переключателем на иврит (HE) и английский
(EN). Идейная основа учебной программы — учение Любавичского Ребе (Рабби Менахем
Мендел Шнеерсон); в шапке присутствует баннер «Yechi» на иврите.

Текущее состояние — **дизайн-прототип / мокап**. Фотографии заменены
плейсхолдерами (`ImgPlaceholder`), часть данных демонстрационная (доноры, суммы
сбора и т.п.). Прототип служит **источником правды по контенту и дизайну** для
переноса на новый стек.

## 2. Целевой стек (продакшен) — РЕШЕНО

Проект переходит с браузерного React-прототипа на полноценную архитектуру.

| Слой | Решение | Примечание |
| --- | --- | --- |
| **Фронт** | **Next.js** (App Router) | SSR/SSG, серверные/клиентские компоненты |
| **Бэк** | **Express** | Отдельный API-сервер |
| **БД** | **MySQL** | Текущие JSX-массивы → таблицы + сиды |
| **Структура** | **Монорепо** | pnpm workspaces (+ опц. Turborepo) |
| **CSS** | **Tailwind** (сайт) | С прокидыванием существующих CSS-переменных |
| **Палитра** | **Только `electric`** | Прочие 3 палитры удаляются; light/dark остаётся |
| **Шрифт (RU/EN)** | **Manrope (Grotesk)** | Латиница и кириллица |
| **Шрифт (HE)** | **Alef** | Иврит, через `:lang(he)` / `[dir="rtl"]` |
| **Админка** | **Отдельное приложение** | CRUD-редактирование контента страниц |

### Почему так

- **Tailwind на сайте, MUI в админке.** В проекте кастомная дизайн-система на
  CSS-переменных. Tailwind читает их через `tailwind.config`
  (`colors: { primary: 'var(--primary)' }`), сохраняя light/dark по
  `data-theme`. Для **админки** удобнее MUI: готовые таблицы, формы, дата-пикеры,
  rich-text — быстрее собрать CRUD, чем верстать руками.
- **Палитра — только `electric`.** Из 4 палитр оставляем одну (синий + золото:
  `--primary` `#5b7dff`/`#2547d6`, `--accent` `#f5c451`). Палитры `stone`,
  `academic`, `minimal` и их переключатель из панели Tweaks **удаляются**.
  Режимы **light/dark сохраняются** (`data-theme`).
- **Два шрифта по языку.** Латиница и кириллица — **Manrope (Grotesk)**; иврит —
  **Alef**. Подключение раздельное: основной `--font-display: "Manrope"`, а для
  ивритского контента шрифт переопределяется через `:lang(he)` и/или
  `[dir="rtl"]` на Alef. Прочие наборы (Unbounded/Cormorant/Playfair) и выбор
  шрифта в Tweaks удаляются. Баннер «Yechi» и любой HE-текст идут под Alef + RTL.
- **Админка — обязательна.** Контент страниц (команда, ученики, программа,
  расписание, распорядок, тексты, кампания пожертвований) редактируется через
  отдельное защищённое приложение. Это делает MySQL единственным источником
  контента, а JSX-массивы прототипа — лишь стартовым сидом.
- **Express — осознанный выбор.** Отдельный API оправдан разделением фронт/бэк и
  тем, что его потребляют **и публичный сайт, и админка**. Один бэк обслуживает
  оба фронта.

### Целевая структура монорепо

```
yeshiva-tkoa/
├─ apps/
│  ├─ web/                  # Next.js — публичный сайт
│  │  ├─ app/               # App Router: home, team, students, study/*, donate, contacts
│  │  ├─ components/        # перенос ui.jsx + pages-*.jsx
│  │  ├─ lib/               # api-клиент, хелперы, провайдер темы
│  │  └─ tailwind.config.ts
│  ├─ admin/                # Next.js + MUI — админка (CRUD контента)
│  │  ├─ app/               # вход, дашборд, экраны редактирования сущностей
│  │  └─ lib/               # api-клиент, авторизация
│  └─ api/                  # Express — общий бэк для web и admin
│     └─ src/
│        ├─ routes/         # public (GET) + admin (CRUD, защищённые)
│        ├─ controllers/
│        ├─ middleware/     # auth (JWT/сессия), валидация
│        ├─ db/             # подключение MySQL, миграции, сиды
│        └─ index.ts
├─ packages/
│  ├─ types/                # общие TS-типы: Teacher, Student, CurriculumSubject, ScheduleSlot, Donation
│  ├─ ui/                   # (опц.) переиспользуемые компоненты
│  └─ config/               # общие eslint / tsconfig / prettier
├─ package.json             # workspaces
├─ pnpm-workspace.yaml
└─ turbo.json               # (опц.) Turborepo
```

`packages/types` — главный смысл монорепо здесь: единый источник типов между
Express и Next, чтобы форма ответа API и UI не расходились.

### Что во что превращается при переезде

- **Данные** (`TEAM`, `STUDENTS`, `CURRICULUM`, `DAILY_BLOCKS`, `SCHEDULE`, доноры)
  → таблицы MySQL (`teachers`, `students`, `curriculum_subjects`, `schedule_slots`,
  `daily_blocks`, `donations`, `donors`, `contact_messages`, опц. `site_content`).
  Текущие массивы становятся сид-данными.
- **Компоненты** (`Home`, `Team`, `Curriculum`, `Schedule`, `Donate` …) → серверные/
  клиентские компоненты Next; логика сохраняется, меняются импорты.
- **Глобальный `window`-неймспейс** и **Babel-в-браузере** → настоящие ES-импорты и
  нормальная сборка.
- **Тема** (CSS-переменные + `data-*`) → сохраняется; Tailwind лишь читает переменные,
  панель Tweaks становится клиентским провайдером темы (без выбора шрифта).
- **Плейсхолдеры** → `next/image` с реальными фото из `uploads/`.
- **Локализация RU/HE/EN** (сейчас декоративная) → `next-intl`/`next-i18next` с
  поддержкой RTL для иврита.

## 3. Текущий прототип — стек (источник, мигрируем ОТ него)

- **React 18.3.1** (UMD с unpkg CDN).
- **Babel Standalone 7.29** — JSX компилируется в браузере, без шага сборки.
- **Чистый CSS** с CSS-переменными; без CSS-фреймворков.
- Шрифты — Google Fonts (в новом стеке остаётся только Manrope).
- Состояние и роутинг — на хуках React, без react-router.

> Эти решения уходят при миграции. Раздел оставлен, чтобы понимать исходную точку.

## 6. Дизайн-система

Тема — атрибут `data-theme` на `<html>` (light/dark). **Палитра одна — `electric`**,
переключатель палитр удалён.

### Палитра — ТОЛЬКО electric

| Токен | Light | Dark |
| --- | --- | --- |
| `--primary` | `#2547d6` | `#5b7dff` |
| `--accent` | `#b88419` | `#f5c451` |
| `--bg` | `#fafbff` | `#0a0e1f` |

Палитры `stone`, `academic`, `minimal` и атрибут `data-palette` из прототипа
**удаляются**. Режимы light/dark сохраняются.

### Шрифты — по языку

| Язык | Шрифт | Как подключается |
| --- | --- | --- |
| RU / EN | **Manrope** (Grotesk) | основной `--font-display`, fallback `system-ui` |
| HE (иврит) | **Alef** | переопределение через `:lang(he)` / `[dir="rtl"]` |

- Прочие наборы (Unbounded/Cormorant/Playfair) и выбор шрифта в Tweaks удаляются.
- Моноширинный акцент — JetBrains Mono (класс `.mono`).
- Фирменный нюанс: акцент заголовка героя — не курсив, жирный (700–800).
- HE-контент (включая баннер «Yechi») идёт под Alef + `dir="rtl"`.

### UI-паттерны

`ImgPlaceholder` (заглушки фото → `next/image`), `Btn` (primary/secondary/ghost +
иконка-стрелка), `Logo` («три башни» + намёк на звезду Давида), анимации
`fade-up`/`fade-up-1…5`.

## 8. Режимы вывода прототипа

1. **Сайт** (`index.html`) — интерактивный прототип.
2. **Дека** (`deck.html`) — слайды 1920×1080 (palette academic).
3. **Печать** (`index-print.html`) — `__printMode = true`.
4. **Single-file** (`Yeshiva ChaBaD Tkoa.html`) — весь сайт в одном HTML.

Флаг `window.__printMode`: обычный `render` в `app.jsx` выполняется только если он
НЕ установлен.

   (Express), `packages/types`, конфиги.

4. Перенести компоненты в `apps/web`; зашить Manrope (RU/EN) + Alef (HE); убрать
   выбор шрифта и палитры.
5. Подключить Tailwind, прокинуть переменные палитры `electric` (сохранить light/dark).
6. Собрать админку: вход, дашборд, экраны CRUD по сущностям контента.
7. Заменить плейсхолдеры на `next/image` с реальными фото (загрузка через админку).
8. Реализовать формы (пожертвования, контакты) против Express + MySQL.
9. Локализация HE (RTL, Alef) и EN вместо декоративного переключателя.

## 10. Рабочие нюансы (из прототипа — учесть при переносе)

- Очерёдность скриптов в HTML важна: `tweaks-panel` → `ui` → `pages-*` → `app`
  (в Next заменяется обычными импортами).
- Глобальный `window`-неймспейс (`Object.assign(window, …)`) → ES-импорты.
- Контент = код: тексты/данные в JSX-массивах → выносятся в БД (первый кандидат).
- Все изображения — плейсхолдеры; реальные фото-референсы в `uploads/`.
- Язык: UI на русском; переключатель RU/HE/EN пока декоративный.
- Зависимость от CDN (React/Babel/шрифты) уходит с переходом на сборку Next.
